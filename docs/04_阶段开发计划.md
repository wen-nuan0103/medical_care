# 在线医疗健康平台 —— 阶段开发计划

> 文档版本：v1.0  
> 文档日期：2026-04-27  
> 目标：按阶段完成可验收模块，最终形成完整在线医疗模拟系统。

| 阶段 | 阶段名称 | 核心目标 | 主要模块 | 优先级 |
|------|----------|----------|----------|--------|
| 阶段 0 | 需求与设计 | 完成需求、技术、模块、数据、接口设计 | 文档体系 | P0 |
| 阶段 1 | 基础平台 | 完成用户、角色、医生资料、药品基础数据 | 登录认证、用户、医生、药品 | P0 |

完成模块：

| 模块 | 完成内容 |
|------|----------|
| 文档体系 | 项目概述、需求文档、模块设计、阶段计划、数据设计、前端设计、接口规划、测试验收 |

交付物：

## 3. 阶段 1：基础平台

目标：

1. 完成项目基础结构。
2. 完成登录认证、角色菜单和基础权限。
3. 完成医生、患者、药剂师、管理员基础数据。
4. 完成药品库基础数据维护。

完成模块：

| 模块 | 完成内容 |
|------|----------|
| 登录认证 | 登录、退出、JWT 校验、角色识别 |
| 用户与角色 | 患者、医生、药剂师、管理员资料 |
| 医生资料 | 医生列表、详情、擅长、职称、服务状态 |
| 药品库 | 药品基础信息、库存、价格、医保属性 |
| 管理后台基础 | 用户、医生、药品的基础管理入口 |

前端页面：

| 页面 | 说明 |
|------|------|
| 登录页 | 账号密码登录 |
| 患者首页 | 医生推荐、药品入口、问诊入口 |
| 医生列表/详情 | 选择指定名医 |
| 管理后台 | 用户和药品基础维护 |

验收标准：

1. 不同角色登录后进入不同首页。
2. 患者可以浏览医生列表和医生详情。
3. 管理员可以维护医生、药剂师和药品基础数据。
4. 接口访问能按角色拦截。

## 4. 阶段 2：私人医生与在线问诊

目标：

1. 完成指定名医服务卡购买。
2. 完成次卡、月卡、季卡、半年卡、年卡权益配置。
3. 完成患者与医生一对一实时聊天。
4. 完成聊天时长限制和医生赠送时长。

完成模块：

| 模块 | 完成内容 |
|------|----------|
| 私人医生卡 | 卡型配置、购买订单、有效期、次数和时长 |
| 在线问诊 | 会话创建、消息发送、历史记录、问诊结束 |
| 时长控制 | 按卡型扣减时长，时长不足提醒 |
| 赠送时长 | 医生赠送额外聊天时长并记录原因 |

前端页面：

| 页面 | 说明 |
|------|------|
| 医生服务卡购买页 | 展示卡型、价格、权益 |
| 我的私人医生 | 查看已购买医生和剩余权益 |
| 问诊聊天窗 | 聊天、计时、处方入口、病史侧栏 |
| 医生接诊工作台 | 查看当前会话、患者信息、赠送时长 |

验收标准：

1. 患者购买服务卡后能看到有效期、剩余次数和剩余时长。
2. 只有拥有有效服务卡的患者能发起问诊。
3. 聊天窗口能实时收发消息。
4. 问诊时长达到限制时系统提醒并限制继续使用。
5. 医生赠送时长后患者剩余时长正确增加。

## 5. 阶段 3：电子处方与药剂师审核

目标：

1. 完成医生在问诊中开具电子处方。
2. 完成处方提交与状态流转。
3. 完成药剂师审核、库存检查和药物相互作用检查。
4. 完成审核意见和患者通知。

完成模块：

| 模块 | 完成内容 |
|------|----------|
| 电子处方 | 诊断、药品、剂量、频次、疗程、用药说明 |
| 库存提示 | 开方和审核时提示库存不足 |
| 药物相互作用 | 根据规则表检查药物组合风险 |
| 药剂师审核 | 通过、驳回、需修改、审核意见 |
| 审计日志 | 记录处方提交和审核操作 |

前端页面：

| 页面 | 说明 |
|------|------|
| 医生开方页 | 从聊天窗口进入，选择药品并填写剂量 |
| 处方详情页 | 患者、医生、药剂师查看处方状态 |
| 药剂师审核台 | 待审处方、风险提示、审核意见 |
| 药物规则管理 | 维护药物相互作用规则 |

验收标准：

1. 医生能为问诊会话开具处方。
2. 药品库存不足时开方页或审核页能提示。
3. 多种药物存在相互作用时系统能给出风险提示。
4. 药剂师审核后处方状态正确变化。
5. 审核通过的处方才能进入购药流程。

## 6. 阶段 4：医保购药闭环

目标：

1. 完成模拟医保卡绑定。
2. 完成处方购药订单。
3. 完成医保卡支付、余额扣减和库存扣减。
4. 完成药品商城基础浏览。

完成模块：

| 模块 | 完成内容 |
|------|----------|
| 医保卡 | 卡号、姓名、余额、状态、报销比例 |
| 订单支付 | 处方订单、支付记录、订单状态 |
| 药品商城 | 药品浏览、搜索、处方药限制 |
| 库存扣减 | 支付成功后扣减库存 |

前端页面：

| 页面 | 说明 |
|------|------|
| 医保卡管理 | 绑定、查看余额、查看状态 |
| 处方购药页 | 根据审核通过处方生成订单 |
| 支付确认页 | 展示医保支付金额、余额、药品明细 |
| 药品商城 | 浏览药品和库存状态 |
| 我的订单 | 查看购药订单和支付记录 |

验收标准：

1. 患者能绑定模拟医保卡。
2. 只有审核通过且未过期处方能下单。
3. 医保余额不足时支付失败并提示原因。
4. 支付成功后医保余额、订单状态、药品库存正确变化。
5. 处方药不能脱离处方直接购买。

## 7. 阶段 5：病患跟踪、用药提醒、Spring AI 与优化

目标：

1. 完成购药后的用药提醒。
2. 完成患者健康指标和症状跟踪。
3. 完成医生查看患者长期跟踪记录。
4. 接入 Spring AI，完成问诊摘要、处方解释、药师审核辅助和提醒文案。
5. 完成统计看板和系统体验优化。

完成模块：

| 模块 | 完成内容 |
|------|----------|
| 用药提醒 | 提醒计划、提醒记录、已服药/漏服 |
| 病患跟踪 | 健康指标、症状变化、随访记录 |
| 医生随访 | 医生查看患者趋势并添加随访建议 |
| Spring AI 智能辅助 | 问诊摘要、处方说明、审核辅助、用药提醒文案 |
| 统计看板 | 订单、处方、问诊、库存等统计 |
| 系统优化 | 错误提示、加载状态、表单校验、移动端适配 |

前端页面：

| 页面 | 说明 |
|------|------|
| 用药提醒页 | 今日用药、提醒设置、服药记录 |
| 健康跟踪页 | 指标录入、症状记录、趋势图 |
| 医生患者档案 | 医生查看患者跟踪和用药执行 |
| AI 辅助面板 | 展示 AI 摘要、建议、引用知识和人工确认状态 |
| 管理看板 | 平台数据统计 |

验收标准：

1. 购药后能按处方自动生成用药计划。
2. 患者能收到提醒并标记已服药或漏服。
3. 患者能记录健康指标和症状变化。
4. 医生能查看患者跟踪记录。
5. 医生能生成并确认 AI 问诊摘要。
6. 药剂师能查看 AI 审核辅助提示，但审核结果仍由药剂师提交。
7. 系统完成主要页面的体验优化和最终测试。
