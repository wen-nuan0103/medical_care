# 在线医疗健康平台 —— 测试与验收标准

> 文档版本：v1.0  
> 文档日期：2026-04-27

## 1. 测试目标

1. 验证项目是否完成在线医疗核心业务闭环。
2. 验证患者、医生、药剂师、管理员四类角色权限是否正确。
3. 验证私人医生卡、聊天时长、电子处方、药剂师审核、医保购药、用药提醒等核心规则。
4. 验证模拟数据的状态流转是否一致。

## 2. 测试范围

| 范围 | 是否必测 | 说明 |
|------|----------|------|
| 登录认证与权限 | 是 | 不同角色访问不同功能 |
| 私人医生卡 | 是 | 卡型、购买、有效期、剩余权益 |
| 在线问诊 | 是 | 聊天、时长、赠送时长 |
| 电子处方 | 是 | 开方、药品、剂量、库存提示 |
| 药剂师审核 | 是 | 审核状态、相互作用、库存 |
| 医保购药 | 是 | 处方下单、医保支付、库存扣减 |
| 药品库 | 是 | 药品维护、库存预警 |
| 病患跟踪 | 建议 | 健康指标、医生查看 |
| 用药提醒 | 建议 | 提醒生成、服药反馈 |
| Spring AI 智能辅助 | 建议 | 问诊摘要、处方解释、药师审核辅助 |
| 统计看板 | 可选 | 数据展示是否正确 |

## 3. 阶段验收

### 3.1 阶段 1 验收：基础平台

| 验收项 | 通过标准 |
|--------|----------|
| 登录 | 患者、医生、药剂师、管理员均可登录 |
| 角色菜单 | 不同角色看到不同菜单 |
| 权限拦截 | 患者无法访问药剂师审核接口 |
| 医生资料 | 患者可以搜索并查看医生详情 |
| 药品库 | 管理员或药剂师可以维护药品 |

### 3.2 阶段 2 验收：私人医生与问诊

| 验收项 | 通过标准 |
|--------|----------|
| 卡型购买 | 患者可购买次卡、月卡、季卡、半年卡、年卡 |
| 权益计算 | 购买后有效期、次数、时长正确生成 |
| 发起问诊 | 只有有效卡可以发起问诊 |
| 实时聊天 | 患者和医生能实时收发消息 |
| 时长控制 | 剩余时长不足时提醒，耗尽后限制继续问诊 |
| 赠送时长 | 医生赠送后患者剩余时长增加并有记录 |

### 3.3 阶段 3 验收：处方与药师审核

| 验收项 | 通过标准 |
|--------|----------|
| 医生开方 | 处方包含诊断、药品、剂量、频次、疗程 |
| 库存提示 | 库存不足时能提示 |
| 相互作用检查 | 处方中存在风险药物组合时能提示 |
| 审核通过 | 通过后处方状态为 APPROVED |
| 审核驳回 | 驳回后处方不可购药 |
| 需修改 | 医生可修改后重新提交 |

### 3.4 阶段 4 验收：医保购药

| 验收项 | 通过标准 |
|--------|----------|
| 医保卡绑定 | 患者可绑定模拟医保卡 |
| 处方下单 | 审核通过且未过期处方可生成订单 |
| 余额校验 | 医保余额不足时支付失败 |
| 库存校验 | 库存不足时支付失败 |
| 支付成功 | 医保余额扣减、库存扣减、订单变为 PAID |
| 处方药限制 | 未审核处方不能购买处方药 |

### 3.5 阶段 5 验收：跟踪提醒与优化

| 验收项 | 通过标准 |
|--------|----------|
| 用药计划 | 购药成功后生成用药计划 |
| 用药提醒 | 到时间生成提醒 |
| 服药反馈 | 患者可标记已服药、漏服、稍后 |
| 健康跟踪 | 患者可记录体温、血压、症状 |
| 医生查看 | 医生可查看自己患者的跟踪记录 |
| 趋势图 | 健康指标可用图表展示 |
| AI 问诊摘要 | 医生可生成摘要草稿并确认 |
| AI 审核辅助 | 药剂师可查看 AI 建议，但审核结果必须人工提交 |

## 4. 核心测试用例

### 4.1 私人医生购卡用例

| 步骤 | 操作 | 期望结果 |
|------|------|----------|
| 1 | 患者登录系统 | 进入患者首页 |
| 2 | 搜索呼吸内科医生 | 展示匹配医生 |
| 3 | 进入医生详情 | 展示医生服务卡 |
| 4 | 选择月卡并购买 | 生成服务卡和订单 |
| 5 | 查看我的私人医生 | 显示剩余次数、剩余时长、到期时间 |

### 4.2 在线问诊用例

| 步骤 | 操作 | 期望结果 |
|------|------|----------|
| 1 | 患者点击发起问诊 | 创建问诊会话 |
| 2 | 患者发送文字消息 | 医生端实时收到 |
| 3 | 医生回复消息 | 患者端实时收到 |
| 4 | 系统计时 | 剩余时长减少 |
| 5 | 医生赠送 10 分钟 | 患者剩余时长增加 |
| 6 | 问诊结束 | 保存问诊记录 |

### 4.3 处方审核用例

| 步骤 | 操作 | 期望结果 |
|------|------|----------|
| 1 | 医生从聊天窗口开处方 | 打开处方表单 |
| 2 | 添加多种药品 | 系统展示库存情况 |
| 3 | 提交处方 | 状态变为待审核 |
| 4 | 药剂师打开审核台 | 能看到待审核处方 |
| 5 | 系统检查药物相互作用 | 展示风险提示 |
| 6 | 药剂师审核通过 | 处方变为可购药 |

### 4.4 医保购药用例

| 步骤 | 操作 | 期望结果 |
|------|------|----------|
| 1 | 患者绑定医保卡 | 显示医保卡余额 |
| 2 | 患者选择已通过处方购药 | 生成购药订单 |
| 3 | 点击医保支付 | 校验余额、库存、处方有效期 |
| 4 | 支付成功 | 医保余额和库存扣减 |
| 5 | 查看订单 | 订单状态为已支付 |

### 4.5 用药提醒用例

| 步骤 | 操作 | 期望结果 |
|------|------|----------|
| 1 | 患者购药成功 | 系统生成用药计划 |
| 2 | 到达提醒时间 | 生成站内提醒 |
| 3 | 患者点击已服药 | 提醒状态变为 TAKEN |
| 4 | 患者未处理提醒 | 超时后可标记为 MISSED |
| 5 | 医生查看用药记录 | 能看到患者服药执行情况 |

### 4.6 Spring AI 辅助用例

| 步骤 | 操作 | 期望结果 |
|------|------|----------|
| 1 | 医生在问诊结束后点击生成摘要 | 系统生成问诊摘要草稿 |
| 2 | 医生编辑并确认摘要 | 摘要写入问诊记录 |
| 3 | 医生对处方生成通俗解释 | 系统生成面向患者的用药说明 |
| 4 | 药剂师打开处方审核 | 系统展示 AI 风险提示和参考知识 |
| 5 | 药剂师提交审核结果 | 审核结果由药剂师决定，AI 只作为辅助 |
| 6 | 管理员查看 AI 日志 | 能看到调用场景、输出结果和确认状态 |

## 5. 权限测试

| 测试项 | 期望结果 |
|--------|----------|
| 患者访问药剂师审核接口 | 返回 403 |
| 药剂师访问医生开方接口 | 返回 403 |
| 未登录访问患者订单接口 | 返回 401 |
| 医生查看非自己患者详情 | 返回 403 或空数据 |
| 患者查看他人处方 | 返回 403 |
| 管理员访问基础数据管理 | 允许访问 |
| 患者直接调用药师 AI 审核接口 | 返回 403 |

## 6. 数据一致性测试

| 场景 | 期望结果 |
|------|----------|
| 医保支付成功 | 订单、医保余额、库存、处方状态同步更新 |
| 支付过程中库存不足 | 支付失败，不扣余额 |
| 处方审核驳回 | 不能创建购药订单 |
| 服务卡过期 | 不能发起新的问诊 |
| 聊天时长耗尽 | 不能继续发送普通问诊消息 |
| AI 建议被拒绝 | 不应写入正式处方说明或审核意见 |

## 7. 最终验收标准

项目最终验收需要满足：

1. 能完整演示“购买指定名医私人医生卡 -> 一对一聊天 -> 医生开处方 -> 药剂师审核 -> 医保卡购药 -> 用药提醒 -> 病患跟踪”流程。
2. 患者、医生、药剂师、管理员四类角色页面和权限清晰。
3. 次卡、月卡、季卡、半年卡、年卡区别明确。
4. 聊天时长约束和医生赠送时长可用。
5. 药剂师能检查药品库存和药物相互作用。
6. 医保卡购买为模拟实现，但余额扣减和支付记录完整。
7. Spring AI 能提供问诊摘要、处方解释和药师审核辅助，且关键医疗结论必须人工确认。
8. 文档、接口、数据库设计与最终功能保持一致。
