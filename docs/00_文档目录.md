## 推荐阅读顺序

1. 先阅读 `01_项目概述与技术选型.md`，确认项目边界和技术路线。
2. 再阅读 `02_功能需求文档.md`，明确系统到底要做什么。
3. 接着阅读 `03_模块设计与优先级.md` 与 `04_阶段开发计划.md`，确定先做哪些模块、每阶段完成什么。
4. 开发前阅读 `05_数据库与模拟数据设计.md`、`06_前端Vue设计方案.md`、`07_API接口规划.md`、`09_SpringAI智能辅助功能设计.md`。
5. 每个阶段结束后按照 `08_测试与验收标准.md` 做功能验证。

## 技术约束

1. 前端使用 Vue 完成。
2. 后端使用 Spring Boot，但不使用 Spring Security。
3. 医保、医疗联网数据、短信、支付均采用模拟实现。
4. 接入 Spring AI 作为智能辅助能力，AI 只提供建议和摘要，不替代医生诊断或药剂师审核。
5. 优先完成“买私人医生卡 -> 在线问诊 -> 开处方 -> 药剂师审核 -> 医保购药”的核心闭环。
