# 在线医疗健康平台 —— 前端 Vue 设计方案

> 文档版本：v1.0  
> 文档日期：2026-04-27  
> 前端框架：Vue 3 + Vite + TypeScript

## 1. 前端目标

1. 完成患者端、医生端、药剂师端、管理后台四类角色页面。
2. 支持私人医生购买、在线聊天、电子处方、药剂师审核、医保购药、病患跟踪、用药提醒。
3. 保证页面流程清晰，适合课程项目展示和答辩演示。
4. 前端仅调用后端 API，不在前端写死关键业务规则。

## 2. 技术栈

| 技术 | 用途 |
|------|------|
| Vue 3 | 页面开发 |
| Vite | 构建和开发服务 |
| TypeScript | 类型约束 |
| Vue Router | 页面路由 |
| Pinia | 全局状态管理 |
| Element Plus | UI 组件 |
| Axios | API 请求 |
| WebSocket | 在线聊天和消息推送 |
| ECharts | 健康趋势和统计图 |
| Day.js | 时间格式化、有效期计算 |

## 6. 状态管理

| Store | 保存内容 |
|-------|----------|
| `authStore` | Token、当前用户、角色、菜单 |
| `chatStore` | 当前会话、消息列表、剩余时长、连接状态 |
| `notificationStore` | 站内信、处方审核通知、提醒通知 |
| `doctorStore` | 医生列表筛选条件、医生详情缓存 |
| `cart/orderStore` | 购药订单、服务卡订单、支付状态 |
| `aiStore` | AI 生成状态、AI 结果、引用知识、人工确认状态 |

## 7. 权限控制

前端权限控制分两层：

1. 路由守卫：根据用户角色判断是否允许进入路由。
2. 页面按钮：根据权限控制按钮显示，例如审核按钮仅药剂师可见。

前端权限只负责体验控制，真正的权限必须由后端接口拦截完成。

## 8. WebSocket 设计

WebSocket 用于三类场景：

| 场景 | 说明 |
|------|------|
| 在线聊天 | 患者和医生一对一消息实时推送 |
| 审核通知 | 处方审核通过、驳回、需修改通知 |
| 用药提醒 | 到点提醒患者服药 |
| AI 生成完成 | AI 摘要或审核建议生成完成后提醒医生/药剂师 |

连接规则：

1. 登录后建立 WebSocket 连接。
2. 握手请求携带 JWT。
3. 断线后自动重连。
4. 聊天消息发送失败时进入重试或失败状态。
